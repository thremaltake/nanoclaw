### Step 1.4: Create Users

**DONE.** Oracle Cloud difference: `ubuntu` user is the admin (skip creating `andry`).

```bash
# Skip creating admin user — 'ubuntu' already has sudo on Oracle Cloud

# Create the NanoClaw service user (no login, no sudo)
sudo adduser --system --group --home /home/nanoclaw --shell /bin/bash nanoclaw

# Create all directories the nanoclaw user will need
sudo mkdir -p /home/nanoclaw/.ssh
sudo mkdir -p /home/nanoclaw/backups
sudo mkdir -p /home/nanoclaw/.claude
sudo mkdir -p /home/nanoclaw/.config/himalaya

# Set ownership and permissions
sudo chown -R nanoclaw:nanoclaw /home/nanoclaw
sudo chmod 700 /home/nanoclaw/.ssh
sudo chmod 700 /home/nanoclaw/.claude
```

**Why two users?**
- `ubuntu` = you, for logging in and system admin (Oracle Cloud default, has sudo)
- `nanoclaw` = the service account, runs NanoClaw with minimal permissions. If NanoClaw is compromised, the attacker can't sudo or access system-level things.

## Phase 2: Set Up NanoClaw on VPS

### Step 2.1: Fork NanoClaw on GitHub

1. Go to https://github.com/qwibitai/nanoclaw in your browser
2. Click the "Fork" button (top right)
3. This creates your own copy at `https://github.com/YOUR_USERNAME/nanoclaw`

**Why fork?** NanoClaw is designed to be forked and customized. Your fork is yours — you can change anything. You can still pull updates from the original repo later.


### Step 2.3: Authenticate Claude Code


This opens Claude Code. It will ask you to authenticate:
1. It shows a URL — copy it
2. Open the URL in your browser (on your Mac or phone)

After authentication, Claude Code stores your credentials at `/home/nanoclaw/.claude/.credentials.json`. This is what NanoClaw will use.

The dashboard's AI chat feature needs to talk to NanoClaw on the VPS. Cloudflare Tunnel creates a secure connection without opening ports.

**On the VPS:**
```bash
# Install cloudflared
curl -fsSL https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update
sudo apt install -y cloudflared

# Authenticate with Cloudflare
cloudflared tunnel login
# Opens a URL — authorize in your browser

# Create a tunnel (this auto-generates a credentials JSON file)
cloudflared tunnel create nanoclaw-ai
# Note the tunnel ID it gives you (a UUID like a1b2c3d4-...)
# It also creates: /home/ubuntu/.cloudflared/YOUR_TUNNEL_ID.json (auto-generated)

# Configure the tunnel
mkdir -p /home/ubuntu/.cloudflared
nano /home/ubuntu/.cloudflared/config.yml
```

Add (replace YOUR_TUNNEL_ID with the UUID from the previous command):
```yaml
tunnel: YOUR_TUNNEL_ID
credentials-file: /home/ubuntu/.cloudflared/YOUR_TUNNEL_ID.json

ingress:
  - hostname: ai.yourdomain.com
    service: http://localhost:3001
  - service: http_status:404
```

**What this does:** Creates a secure tunnel from Cloudflare to your VPS. The dashboard calls `ai.yourdomain.com` which Cloudflare routes to NanoClaw's AI endpoint on the VPS. No ports opened on your firewall.

**Note:** You'll need a domain name on Cloudflare for this. If you don't have one yet, you can skip this step and add it later — the dashboard will work for everything except AI chat.

## Phase 7: Set Up systemd Service

Once testing passes, set up NanoClaw as a proper service that auto-starts.

### Step 7.1: Build for Production

```bash
sudo -u nanoclaw -i
cd /home/nanoclaw/nanoclaw
npm run build
# This compiles TypeScript to JavaScript in /home/nanoclaw/nanoclaw/dist/
```

### Step 7.2: Create systemd Service

```bash
# As your admin user (not nanoclaw)
sudo nano /etc/systemd/system/nanoclaw.service
```

Add:
```ini
[Unit]
Description=NanoClaw AI Assistant
After=docker.service network-online.target
Wants=network-online.target
Requires=docker.service

[Service]
Type=simple
User=nanoclaw
Group=nanoclaw
WorkingDirectory=/home/nanoclaw/nanoclaw
ExecStart=/usr/bin/node dist/index.js
Restart=always
RestartSec=10
Environment=NODE_ENV=production

# Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=read-only
ReadWritePaths=/home/nanoclaw/nanoclaw/store
ReadWritePaths=/home/nanoclaw/nanoclaw/groups
ReadWritePaths=/home/nanoclaw/.claude

[Install]
WantedBy=multi-user.target
```

**What each line does:**
- `After=docker.service` — wait for Docker to start first
- `User=nanoclaw` — run as the limited service user
- `Restart=always` — if it crashes, restart it
- `RestartSec=10` — wait 10 seconds before restarting
- `NoNewPrivileges=true` — process can't gain extra permissions
- `ProtectSystem=strict` — system directories are read-only
- `ProtectHome=read-only` — can't write to other users' homes
- `ReadWritePaths=...` — only these specific directories are writable

```bash
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable nanoclaw
sudo systemctl start nanoclaw

# Check it's running
sudo systemctl status nanoclaw

# Watch logs
sudo journalctl -u nanoclaw -f
```

### Step 7.3: Add Orphan Container Cleanup

```bash
# As admin user
sudo crontab -e
```

Add:
```
# Every hour, clean up any orphaned NanoClaw containers older than 2 hours
0 * * * * docker ps -q --filter "name=nanoclaw-" --filter "status=running" | xargs -r docker inspect --format '{{.Id}} {{.State.StartedAt}}' | awk -v cutoff=$(date -d '2 hours ago' +\%s) '{cmd="date -d "$2" +\%s"; cmd | getline ts; close(cmd); if (ts < cutoff) print $1}' | xargs -r docker rm -f 2>/dev/null
```
